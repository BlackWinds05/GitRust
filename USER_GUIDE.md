## 1. 总览：侧边栏导航

进入任意仓库后，左侧边栏是你操作的主要入口：

```
📁 Files         → 浏览代码文件
🕐 Commits       → 查看提交记录
⚠️ Issues        → 任务/缺陷管理
⇄ Merge Requests → 分支合并
🎯 Milestones    → 里程碑进度
🏷️ Labels        → 标签管理
👥 Members       → 成员权限
📖 Wiki          → 知识文档
🌐 Graph         → 分支可视化
📊 Stats         → 数据统计
⚙️ Settings      → 仓库设置
```

## 2. Files — 代码文件浏览与编辑

**对应 GitHub：** Repository Code 页面

### 你能做什么

| 操作 | 怎么做 |
|------|--------|
| 浏览目录结构 | 点击文件夹进入，面包屑导航返回上级 |
| 查看文件内容 | 点击文件名，显示代码 |
| 切换分支查看 | 顶部下拉选分支 |
| 在线编辑文件 | 文件页面点 **Edit File** → 修改 → 填提交信息 → Commit |
| 创建新文件 | 目录页点 **New File** → 填文件名+内容 → 选分支 → Create |
| 查看 README | 仓库首页自动渲染 README.md |

### 当前状态
- ✅ 目录浏览、文件查看、README渲染
- ✅ 在线编辑和新建文件
- ⚠️ 暂不支持文件删除/重命名（Web UI 层面）
- ⚠️ 代码暂未语法高亮

## 3. Commits — 提交历史

**对应 GitHub：** 仓库 Commits 页面

### 你能做什么

| 操作 | 怎么做 |
|------|--------|
| 查看提交列表 | 每条显示作者、时间、SHA、提交信息 |
| 查看提交详情 | 点击提交信息 → 看到完整 diff（红删绿增） |
| 翻页浏览 | 底部页码导航 |

### 提交详情页
- **提交信息** — 作为标题
- **SHA 值** — 完整的 commit hash
- **父提交** — 显示 parent SHA
- **文件变更** — 列出本次修改了哪些文件
- **Diff 对比** — 逐行展示代码增删，红色=删除，绿色=新增

## 4. Issues — 任务与缺陷追踪

**对应 GitHub：** Issues 功能

这是你在项目中管理任务、Bug、需求的地方。

### 创建 Issue

点击 **New Issue**，填写：

| 字段 | 说明 | 必填 |
|------|------|------|
| Title | 标题 | ✅ |
| Description | Markdown 描述 | ❌ |
| Due Date | 截止日期 | ❌ |
| Labels | 打标签分类 | ❌ |
| Milestone | 归属哪个里程碑 | ❌ |
| Assignees | 分配给谁 | ❌ |

### Issue 列表

- **Open / Closed** 标签切换
- 筛选栏：按关键词搜索、按里程碑筛选、按标签筛选
- 每条 Issue 显示：状态图标、标题、编号、作者、时间、标签数量

### Issue 详情

- 查看完整描述（Markdown 渲染）
- **操作按钮：** Close/Reopen、Edit、Delete
- **评论区：** 发表评论（Markdown 格式），可删除自己的评论
- 显示：标签、里程碑、负责人、截止日期

### 当前状态
- ✅ 创建、编辑、删除、关闭/重开
- ✅ 分配标签/里程碑/负责人/截止日期
- ✅ 评论系统
- ✅ 筛选（关键词、里程碑、标签）
- ⚠️ 筛选结果不回显（刷新后可见但表单不保留选择值）

## 5. Merge Requests — 合并请求

**对应 GitHub：** Pull Requests

当你想把一个分支的代码合并到另一个分支时使用。

### 创建 MR

1. 点击 **New Merge Request**
2. 选择 **源分支**（你要合并的代码在哪个分支）
3. 选择 **目标分支**（要合并到哪个分支，通常是 main）
4. 填标题和描述
5. 点击 Create

### MR 详情

- 显示源/目标分支
- **Diff 对比：** 展示两个分支的代码差异
- **冲突检测：** 如有冲突，红色警告框提示
- **操作按钮：** Merge（合并）、Close（关闭）

### 当前状态
- ✅ 创建、查看 diff、合并、关闭
- ✅ 冲突检测提示
- ⚠️ 暂不支持 MR 评论
- ⚠️ 创建时不显示变更预览（创建后才看到 diff）

## 6. Milestones — 里程碑

**对应 GitHub：** Milestones

用于规划版本迭代，跟踪整体进度。

### 创建里程碑

在列表页顶部表单，填**标题**和**截止日期**即可。

### 里程碑列表

- 每个里程碑卡片显示：标题、状态（open/closed）、进度条
- **进度条：** 绿色 = Issues 已完成比例，灰色背景 = 总数
- 显示 "N closed / M open" 文字

### 里程碑详情

点击里程碑标题进入：

- 基础信息 + 进度条
- **已绑定 Issues 列表** — 可逐个解绑
- **添加 Issue** — 下拉选择未绑定 Issue 进行关联

### 操作按钮
- **Edit：** 修改标题/描述/截止日期
- **Close/Reopen：** 切换状态
- **Delete：** 删除（会解绑所有关联 Issue）

### 当前状态
- ✅ 创建、编辑、切换状态、删除
- ✅ 关联 Issues、进度条
- ✅ 详情页绑定/解绑 Issues

## 7. Labels — 标签管理

**对应 GitHub：** Labels

用于给 Issues 分类标记（如 "bug"、"enhancement"、"urgent"）。

### 创建标签

在列表页顶部填写：
- **Name：** 标签名
- **Color：** 颜色（十六进制，如 `#ff0000`）
- **Description：** 可选描述

### 标签列表

每个标签以彩色卡片展示，名称用对应颜色。

### 编辑 / 删除标签
- 每个标签卡片有 **Edit** 按钮 → 点击弹出内联编辑表单
- 每个标签卡片有 **X** 按钮 → 点击确认后删除

### 当前状态
- ✅ 创建、编辑、删除标签（需登录）
- ⚠️ 颜色输入为纯文本 hex 值（暂无取色器）

## 8. Members — 成员管理

**对应 GitHub：** Settings > Collaborators

管理谁可以访问这个仓库。

### 添加成员

填 **用户名**，选 **权限级别**：

| 权限 | 能做 |
|------|------|
| Read | 查看代码和页面 |
| Write | 查看 + 推送代码 |
| Admin | 查看 + 推送 + 管理设置 |

### 修改权限 / 移除成员
- 权限栏是**下拉框**，切换后自动提交修改
- 最右有 **Remove** 按钮，点击确认后移除

### 成员列表

显示用户头像（首字母）、名称、权限（可修改）、添加时间。

### 当前状态
- ✅ 添加、修改权限、移除成员（需登录）
- ⚠️ 暂未显示真实头像图片（显示用户名首字母）

## 9. Wiki — 项目知识库

**对应 GitHub：** Wiki

写项目文档、使用说明、开发规范的地方。

### 创建页面

1. 点击 **New Page** → 进入编辑器
2. 填写标题和内容（Markdown 格式）
3. 点击 Save

### 查看页面

- 首页显示 Home 页面内容（如果有的话）
- 下方列出所有已创建的页面

### 页面操作

- **Edit：** 修改内容（每次保存创建新版本）
- **History：** 查看所有历史版本
- **Rollback：** 回滚到某个历史版本
- **Delete：** 删除页面
- **Search：** 右上角搜索框，按标题和内容检索

### 当前状态
- ✅ 创建、编辑、删除、搜索
- ✅ 版本历史与回滚
- ⚠️ "New Page" 按钮默认创建 Home 页面；创建自定义页面需要手动改 URL

## 10. Network Graph — 分支网络图

**对应 GitHub：** Insights > Network

可视化展示所有分支及其提交关系。

- 每个圆点 = 一个提交
- 不同颜色 = 不同分支
- **点击节点** → 跳转查看该提交详情
- 右侧显示对应的提交列表

### 当前状态
- ✅ 可视化渲染
- ✅ 可点击跳转
- ⚠️ 暂不支持缩放/拖拽

## 13. 常见操作场景

### 场景1：创建一个 Bug Issue 并跟踪

1. 进入 Issues → **New Issue**
2. 标题写 "登录页面崩溃"
3. 描述里写复现步骤
4. 选 Label `bug`（红色）
5. 分配给成员 `@dev1`
6. 设截止日期
7. 创建后，成员可以在评论区讨论
8. 修完后 Close

### 场景2：提交代码并创建合并请求

1. 本地 `git checkout -b fix-bug`
2. 修改代码，`git commit`，`git push`
3. 到 Merge Requests → **New MR**
4. Source: `fix-bug`，Target: `main`
5. 查看 diff 确认改动
6. 无冲突 → 点击 **Merge**

### 场景3：规划版本迭代

1. 到 Milestones → 创建 "v1.0 发布"
2. 设截止日期 `2026-07-01`
3. 把相关 Issues 在详情页逐一 Bind
4. 进度条会自动更新
5. 全部 closed → 切 Close 状态 → 发布

### 场景4：编写项目文档

1. 到 Wiki → 点 New Page
2. 标题 "开发环境搭建"
3. 内容用 Markdown 写步骤
4. Save → 团队成员可阅读
5. 后续可在 History 中回滚到旧版本

## 给开发组成员的备注

如果你在测试或开发过程中遇到报错：

1. **"Expected N but got M"** → 通常是路径参数不匹配，需要检查 handler 的 `Path<T>` 和路由的 `{param}` 数量是否一致
2. **空白页或 `--` 占位符** → 后端没有传入对应的模板变量
3. **表单提交后闪退或没反馈** → 可能缺少 flash message（`helpers::flash`）

完整的技术文档见 [RUNBOOK.md](RUNBOOK.md)（部署指南）和本文件的模块说明。
